Etiquetas <insert>,<select> & <textarea>

Etiqueta Descripcion Ejemplo
<input type="text"> Representa a un campo de entrada de texto. El control asociado a este campo es una caja de texto que permite a los usuarios editar una sola línea de texto regular. Los controles de texto son útiles para recolectar líneas cortas de texto como títulos, nombres, apellidos, descripciones cortas, palabras clave, etiquetas, etc.

Nombre:

<input type="password"> Proporcionan una forma para que el usuario ingrese una contraseña de forma segura. El elemento se presenta como un control de editor de texto, sin formato, de una línea, en el que el texto está oculto para que no pueda leerse.

Contraseña:

<input type="hidden"> El atributo global hidden es un atributo Booleano que indica que el elemento todavía no está , o ya no es relevante . Por ejemplo , puede ser usado para ocultar elelementos de la página que no pueden ser usados hasta que el proceso de login se haya completado. El explorador no dibujará dichos elementos.

Nombre:

<input type="tel"> Representa un campo para un número telefónico. El control asociado a este campo es una caja de texto que permite a los usuarios editar una sola línea de texto regular, sin requerimientos partiuclares sobre el formato.

Telefono:

<input type="search"> Representa un campo de texto de una sola línea para propósitos de búsqueda. El control asociado a este campo es una caja de texto que permite a los usuarios editar una sola línea de texto regular. Los controles de búsqueda son útiles para recopilar líneas cortas de texto que los agentes procesadores o programas del lado cliente usarán como parámetros para conducir operaciones de búsqueda.

Buscar:

<input type="email"> se utilizan para permitir que el usuario ingrese y edite una dirección de correo o, si se especifica el atributo multiple, una lista de direcciones de correo.

Correo:

<input type="date"> Crean campos de entrada que permiten a los usuarios proporcionar una fecha, bien a través de un cuadro de texto que valida automáticamente el contenido, o bien mediante una interfaz especial de selección de fechas.

Fecha:

<input type="url"> Representa un campo para un solo URL absoluto. El control asociado a este campo es una caja de texto que permite a los usuarios editar una sola línea de texto regular.

URL:

<input type="color"> Proporciona un elemento de interfaz de usuario que permite a los usuarios especificar un color.

color:

<input type="range"> Permite que el usuario especifique un valor numérico comprendido entre un valor mínimo y máximo.

Volumen:

<input type="radio"> Representa una opción que pertenece a un grupo en el que no más de una opción puede ser seleccionada al mismo tiempo.

radio: Masculino
<input type="number"> Son usados para permitir al usuario ingresar un número. Éstos incluyen validación incorporada para rechazar entradas no numéricas.

Numeros:

<input type="image"> El atributo “image” sirve para personalizar un botón. Es decir, este atributo introduce un botón en una imagen, sustituyendo al formato estándar de botones.

<input type="file"> Representa un control para seleccionar una lista de uno o más archivos para ser subidos al servidor. Cuando el formulario es enviado, los archivos seleccionados son subidos al servidor, junto con su nombre y tipo.

Archivo:
<input type="button"> Que se usa para crear un botón en el que se puede hacer click sin ningún valor por defecto, es decir, no tiene un comportamiento predeterminado.

<input type="datatime-local"> Representa un campo para una fecha y hora local sin información de zona horaria. Este tipo de campos es particularmente útil para recolectar un punto en el tiempo, que es específco de un lugar.

Tiempo local:

<input type="month"> Representa un campo para la entrada de un mes. En los navegadores que soportan el mecanismo, estos campos pueden estar representados por controles que permiten a los usuarios cambiar su valor de manera gráfica.

Mes:

<input type="time"> representa un campo para la entrada de una hora.

hora:

<input type="reset"> Representa un botón que, cuando es presionado, restaura todos los campos del formulario (form) al cual pertenece, a sus valores iniciales.

<input type="submit"> Sirve para enviar el formulario una vez el usuario ha rellenado todos sus campos y pulsa en este botón.

<select form> Representa un formulario. En este formulario se agregan los diferentes campos de entrada de datos o de confirmación, así como los botones, que como mínimo ha de haber uno, el de envío.
<select disabled> Nos propone, esta vez para desactivar campos al menos temporalmente.
<select autofocus> La función de este atributo es poner el cursor de manera activa en un input del formulario sin necesidad de hacer click en él.
<select name> se usa para dar el nombre a un control y permite acceder a su contenido.
<select size> El atributo size especifica el número de opciones visibles en una lista desplegable.
<select multiple> Representa un control para seleccionar cero o más opciones de la lista.
<textarea name> El atributo name especifica un nombre para un área de texto. El atributo name se utiliza para hacer referencia a elementos de JavaScript o para hacer referencia a datos de formulario después de enviar un formulario.
<textarea autofocus> El atributo autofocus es un atributo booleano. Cuando está presente, especifica que el área de texto debe obtener automáticamente el foco cuando se carga la página. ¿Qué haces?:

<textarea required> Este atributo indica que el usuario debe rellenar el contro con un valor antes de poder enviar el formulario. Envia pensamiento:

<textarea maxlength> Indica el número máximo de caracteres (Unicode code points) que el usuario puede insertar. Si no está especificado entonces el usuario puede insertar un número ilimitado de caracteres.
<textarea minlength> El mínimo número de caracteres (Unicode code points) que el usuaurio debe insertar.
